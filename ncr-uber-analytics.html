About this project

This dashboard was built as a full end-to-end data analytics project to practice turning raw ride-booking data into actionable insights. Step 1: Data Sourcing I started with a CSV dataset (ncr_ride_bookings.csv) that included booking details such as ride distance, payment method, vehicle type, status (completed, cancelled, incomplete), and customer/driver ratings. Step 2: Data Preparation The dataset was cleaned and standardized inside the app using PapaParse. This step made sure missing values, nulls, and inconsistent labels didn’t break the analysis. Step 3: Dashboard Design I designed the dashboard layout based on real-world analytics needs: KPIs (bookings, revenue, ratings, completion rate) Trends over time (bookings, revenue) Vehicle type & payment method distributions Cancellation patterns and reasons Ratings breakdowns Top routes by frequency and value Auto-generated insights and summaries Step 4: Implementation The dashboard itself was coded in React + Vite, with: Recharts for interactive charts Tailwind CSS for styling and layout lucide-react for icons React state hooks for filters (vehicle type, payment method, date ranges)
