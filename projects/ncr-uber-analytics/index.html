About this project

This dashboard was built as a full end-to-end data analytics project to practice turning raw ride-booking data into actionable insights.

Details:
Step 1: Create tables and import CSVs

Build database tables and bring in raw CSV data for analysis.

  • Primary file: ncr_ride_bookings.csv
  • Key fields: ride distance, payment method, vehicle type, status (completed, cancelled, incomplete), customer & driver ratings
Step 2: Data Preparation

Clean, standardize, and format data to ensure accuracy and consistency.

  • Parsed and cleaned in-app using PapaParse
  • Handled missing values and nulls
  • Standardized labels/data types to prevent breakage during analysis
Step 3: Dashboard Design

Structure the dashboard layout to highlight KPIs, trends, and insights.

  • KPIs (bookings, revenue, ratings, completion rate)
  • Trends over time (bookings, revenue)
  • Vehicle type & payment method distributions
  • Cancellation patterns and reasons
  • Ratings breakdowns
  • Top routes by frequency and value
  • Auto-generated insights and summaries
Step 4: Implementation

Develop and deploy the dashboard with interactive, user-ready features.

The dashboard was coded in React + Vite, using:

  • Recharts for interactive charts
  • Tailwind CSS for styling and layout
  • lucide-react for icons
  • React state hooks for filters (vehicle type, payment method, date ranges)
